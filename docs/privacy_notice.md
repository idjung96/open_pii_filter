# 개인정보 처리방침 (Privacy Notice)

> Phase 6 산출물 — ISMS-P / 개인정보 보호법 § 30(개인정보 처리방침의 수립 및 공개) 대비 템플릿.
> 운영 시점에는 `{{COMPANY}}`, `{{CONTACT}}` 등 자리표시자(placeholder)를 실제 값으로 치환합니다.

---

## 1. 개인정보의 처리 목적

`{{COMPANY}}`(이하 "회사")는 게시판 게시 전 단계에서 **본인 또는 제3자의 개인정보 노출을 사전 차단**하기 위해 PII 탐지·마스킹 API(이하 "본 API")를 운영합니다. 처리 목적은 다음과 같습니다.

1. 게시 본문(post.body)·제목(post.title)에 포함된 PII 자동 식별
2. 첨부파일(PDF/DOCX/HWP/HWPX/이미지) 본문 PII 식별
3. 식별된 PII가 포함된 게시 시도의 거부(BLOCK) 처리
4. 보안 사고 대응을 위한 감사로그(audit log) 기록

본 API는 게시 전(pre-publication) 자동 검사기로서 **수집된 PII를 별도 보존하지 않으며**, 검사 결과 메타데이터만 일정 기간 보관합니다.

## 2. 처리하는 개인정보 항목

| 구분 | 항목 | 비고 |
|------|------|------|
| 입력값 | 게시 본문 / 제목 / 첨부파일 텍스트 | **즉시 처리·즉시 폐기** (저장하지 않음) |
| 메타데이터 | request_id, api_key_id, 요청 IP, 처리 결과 코드, 검출 항목 유형 | DB에 저장 (감사로그) |
| 검출 결과 | start/end 위치, entity_type, score | **원문 PII 평문은 저장하지 않음** |

> **저장하지 않는 항목**
> - PII 평문 (주민등록번호, 전화번호, 이메일, 주소, 카드번호 등)
> - 원본 첨부파일 바이너리
> - 요청 본문 전체

## 3. 처리 및 보유 기간

| 자료 | 보존 기간 | 근거 |
|------|----------|------|
| 검출 결과 메타데이터 (`extraction_jobs`) | 30일 (`Settings.detection_retention_days`) | 운영상 필요 최소 |
| 감사로그 (`audit_events`) | 1년 (`Settings.audit_log_retention_days`) | ISMS-P 권고, 정보통신망법 § 28의2 |
| API 키 (`api_keys.secret`) | 폐기 시까지 (`revoke` 명령) | 키 관리 정책 |

만료된 모든 자료는 자동 GC 워커에 의해 영구 삭제됩니다.
- `app.workers.audit_cleanup` (1시간 주기)
- `app.workers.job_cleanup` (1시간 주기)

## 4. 개인정보의 제3자 제공

본 API는 **제3자에게 개인정보를 제공하지 않습니다.**

- 외부 통신: 호출자가 지정한 `callback_url`로 탐지 결과 메타데이터만 전송 (PII 평문은 포함되지 않습니다).
- 외부 OCR / VLM 엔드포인트는 회사 내부망(사내 vLLM)을 사용하며 외부 클라우드로 송출되지 않습니다.

## 5. 처리 위탁

본 API는 외부 처리위탁을 수행하지 않습니다. 모든 처리는 회사 내부 인프라에서 이루어집니다.

## 6. 정보주체의 권리·의무 및 행사 방법

게시 전 검사 단계에서 PII가 검출되어 게시가 차단된 경우, 정보주체는 회사가 정한 절차에 따라 다음 권리를 행사할 수 있습니다.

1. 본인 게시물의 검사 결과 및 차단 사유 확인 요청
2. 잘못된 차단 결과(오탐)에 대한 이의 신청
3. 감사로그상 본인 식별자(api_key_id) 관련 기록 열람·삭제 요청

문의: `{{CONTACT}}`

## 8. 개인정보의 안전성 확보 조치

| 분류 | 조치 |
|------|------|
| 관리적 조치 | 최소권한 원칙 (`is_admin` 키 분리), 분기별 키 로테이션, 감사로그 1년 보존 |
| 기술적 조치 | AES-256-GCM 컬럼 암호화 (`app.security.encryption`), HMAC-SHA256 요청 서명, IP 화이트리스트, 로그 PII 자동 스크럽 |
| 물리적 조치 | 운영 서버 사내 데이터센터 보관, 접근 통제 |

## 9. 개인정보 처리방침의 변경

본 처리방침은 시행일로부터 적용되며, 법령 및 방침에 따른 변경내용의 추가, 삭제 및 정정이 있을 경우에는 변경 사항의 시행 7일 전부터 공지사항을 통해 사전 공지하겠습니다.

- 시행일: `{{EFFECTIVE_DATE}}`
- 최종 개정일: `{{LAST_UPDATED}}`
